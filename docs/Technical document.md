# 3.1.2 消除深度穿透的多帧密度聚类算法

本文的视觉感知管线采用"目标检测联合关键点提取"的级联策略，在二维图像中输出龙眼主梗的采摘点（像素坐标）。在室外非结构化环境（如光照变化、微风扰动等）中，视觉模型推理出的关键点不可避免地会产生随机的微小像素偏移。由于待采摘的龙眼果梗极细（直径通常仅为 3~8 mm），一旦二维关键点发生哪怕几个像素的偏移，就会轻易滑出果梗的真实物理边界。此时，该像素点对应的深度值会直接"穿透"果梗，错误地提取到后方深邃的背景枝叶深度，导致映射出的三维靶点坐标在果梗表面（如深度 350 mm）与背景树冠（如深度 600 mm）之间产生巨大的跳变误差。

**【此处需补充您的实验图片 1：深度穿透现象图】**

*建议内容*：放一张对比图。左边是 2D 图像，展示关键点稍微偏离果梗边缘一点点；右边是 3D 点云图（或深度图），用红圈标出这个点直接打在了远处的背景树叶上，深度值发生了突变。

正是由于这种由微小像素偏移引发的"深度穿透"现象，使得系统获取到的任意单帧（尤其是初次锁定目标的第一帧）三维坐标都存在极大的随机性与欺骗性。如果直接将第一帧的结果作为真实靶点下发控制指令，一旦该帧恰好发生了深度穿透，机械臂将根据偏差达数十厘米的错误坐标生成逼近轨迹，导致末端夹爪直接刺入树冠深处，引发严重的树体破坏与设备碰撞。

为了彻底消除这种单帧定位误差与撞机隐患，本系统引入了基于 DBSCAN（Density-Based Spatial Clustering of Applications with Noise）的多帧时空密度聚类融合策略。在连续的视频流观测中，绝大多数帧的二维关键点依然能准确命中果梗，真实的果梗坐标在三维空间中必然呈现出高密度的聚集特性；而因偶尔偏移引发的背景穿透点，则呈现为低密度的散乱分布。通过开启多帧时序观测滑窗并应用 DBSCAN 算法，系统能够利用数据的统计学空间密度特征，精准剔除偶尔打在背景上的穿透离群点。具体算法流转流程如下：

**步骤一：构建采摘点时序观测滑窗**

当顶层状态机（HFSM）处于观测态且视觉算法连续输出二维关键点后，系统开启一个容量为 $N$ 的时序滑动窗口（经综合测试平衡实时性与安全性，设定窗口帧数 $N=15$）。结合深度图与手眼标定矩阵，将这 $N$ 帧提取到的二维关键点，统一映射为全局世界坐标系下的三维观测点集 $P = \{p_1, p_2, ..., p_N\}$。

**【此处需补充您的实验图片 2：DBSCAN 聚类去噪效果图】**

*建议内容*：放一张 3D 坐标系下的散点图。展示 $N$ 帧积累下来的点云：前面有一团密集的点（标记为绿色，这是真实果梗簇），后面 Z 轴深处散落着几个零星的点（标记为红色，这是背景穿透噪声）。图注说明 DBSCAN 成功保留了绿色簇、剔除了红色噪声。

**步骤二：基于 DBSCAN 的密度校验与穿透点剔除**

设定 DBSCAN 算法的核心参数：考虑到传感器在真实果梗表面的正常测量容差，设定邻域搜索空间半径 $\epsilon = 15$ mm，核心点最小样本数阈值 $MinPts = 8$（即要求在 15 帧观测中，必须有超过半数的帧准确命中果梗，未发生滑落穿透）。算法遍历三维观测点集 $P$，对于任意采样点 $p_i$，若其半径 $\epsilon$ 的球形邻域内包含的点数 $N_\epsilon(p_i) \geq MinPts$，则将其标记为核心点，并向外扩张形成聚类簇。经过扫描后，那些映射到后方背景枝叶上的"首帧穿透点"或"散乱噪声点"，由于无法满足局部密度阈值要求，将被算法判定为离群噪声（Noise）并予以剔除。系统最终仅保留唯一的高密度核心坐标簇 $C_{core}$，即采摘点的真实三维物理锚点。

**步骤三：加权质心融合与方差收敛校验**

在提取出纯净的核心簇 $C_{core}$ 后，对其内部点集求取三维空间均值，输出最终的绝对稳定定位坐标 $P_{final} = (x, y, z)$。同时，系统并行计算该核心簇的三维坐标均方差 $\sigma$。当且仅当 $\sigma$ 衰减至系统设定的极限安全容差阈值（如 $\sigma < 3$ mm）以内时，系统才判定三维靶点已彻底收敛。此时，触发"目标锁定完毕"事件信号，驱动状态机安全跃迁至下一步的协同预分配状态。

## 项目搭建缘由与工程背景

本项目起源于眼在手上（Eye-in-Hand）机械臂在龙眼园中的真实采摘需求。实际作业中，视觉系统输出的是果梗候选点的三维坐标，而机械臂控制系统默认将该坐标视为可信靶点并执行逼近。问题在于：果梗直径小、背景枝叶密、光照和风扰强，导致二维关键点仅发生数像素偏移就可能滑出果梗，深度值穿透到后景，进而在 $Z$ 方向产生大幅跳变。该跳变会直接放大为末端轨迹偏差，最终造成抓取失败、树体损伤，甚至机械臂碰撞风险。

因此，项目从一开始就不是“提高一点检测精度”的优化问题，而是“保障机械臂作业安全与成功率”的系统级必要能力建设。多帧聚类模块在本系统中承担的是**安全网角色**：它的核心价值是拦截不可靠单帧坐标，避免错误目标进入运动规划链路。

## 项目必要性分析

从系统工程角度，采用多帧密度聚类具有以下必要性：

1. **单帧结果不具备稳定可控性**  
在果园环境中，光照闪烁、叶片摆动、遮挡变化会造成单帧输出高度随机，无法满足工业级“可重复、可验证、可追责”的控制输入要求。

2. **错误坐标的风险成本远高于算法计算成本**  
多做十几帧融合的计算代价毫秒级，而一次错误逼近导致的停机或硬件损伤代价远高于计算开销。

3. **控制系统需要的是“收敛信号”而非“某一帧数值”**  
机械臂状态机更关心“是否可安全锁定目标”，因此除了输出坐标，还必须输出稳定性指标（如 $\sigma$）作为跃迁判据。

4. **多目标场景是常态而非例外**  
相机视野中常同时出现多串龙眼，视觉模块又无法提供稳定 ID，必须引入多目标关联机制，保证每个果串轨迹独立融合。

## 算法选型回顾与取舍依据

为确定最终方案，项目对常见思路进行了工程取舍：

| 方案 | 优点 | 局限 | 结论 |
| --- | --- | --- | --- |
| 单帧直接控制 | 延迟最低 | 对穿透噪声几乎无抵抗 | 不可用 |
| 滑动平均/中值滤波 | 实现简单 | 无法显式识别离群噪声，穿透点会污染均值 | 不足 |
| Kalman / EKF | 对连续运动估计友好 | 依赖噪声模型与状态假设，难覆盖突发穿透离群 | 作为补充，不作主去噪 |
| KMeans | 计算快 | 需预设簇数，且对离群点敏感 | 不适合作为主去噪 |
| DBSCAN | 可直接标注噪声点（`Noise`），无需预设簇数，密度判别直观 | 参数需结合传感器与任务标定 | 最终采用 |

最终选择 DBSCAN 的关键原因是：本问题的本质是“在小样本滑窗内，从含离群点的三维观测中提取高密度真实簇”，DBSCAN与问题结构高度匹配。

## 实现架构与关键设计决策

### 1) 单目标融合链路

- 输入：每帧一个果梗候选 `xyz`
- 处理：滑窗累积（$N=15$）-> DBSCAN -> 保留最大点数簇 -> 计算 $P_{final}$ 与 $\sigma$
- 输出：`locked / p_final / sigma / core_size / reason`

其中“最大点数簇”用于对应本任务的统计假设：真实果梗在多数帧内应被重复观测，形成主簇；穿透点通常稀疏、短时、低密度。

### 2) 多目标扩展（无稳定ID）

在“同帧出现多串龙眼、视觉无稳定ID”的约束下，系统增加了多目标轨迹管理：

- 采用最近邻关联（Nearest Neighbor）进行观测分配
- 采用距离门限控制关联范围（默认 `association_dist_thresh_mm=80`）
- 每条轨迹独立维护一个 DBSCAN 滑窗融合器
- 通过 `max_missed_frames` 管理轨迹生存周期

该设计保证了“每个果串各自融合、互不污染”，满足真实果园多目标作业场景。

## 参数选取依据（可写入论文实验设置）

- **滑窗帧数 $N=15$**：在实时性与稳健性之间折中。帧数过小，统计稳健性不足；帧数过大，引入响应滞后。
- **$\epsilon = 15$ mm**：覆盖果梗局部测量抖动与轻微机械振动，避免将同一目标误判为离散簇。
- **$MinPts = 8$**：要求窗口内超过半数观测支持同一簇，提升抗偶发穿透能力。
- **$\sigma_{th} = 3$ mm**：对应末端抓取对定位稳定性的安全容差门槛。
- **多目标关联门限 80 mm**：用于无ID条件下的同目标连续性判定，兼顾抖动容忍与串间分离。
- **轨迹失配容忍 3 帧**：避免短时丢检导致轨迹频繁重建。

## 典型应用场景

1. 果园户外采摘：光照变化明显、背景纹理复杂、风扰导致目标局部摆动。  
2. 密集果串作业：同视野多目标并存，视觉模块仅给出候选点而无稳定ID。  
3. 安全敏感控制链：末端执行器靠近枝叶区域，错误坐标可能导致碰撞。  
4. 人机协同场景：需要“可解释的锁定判据”向上层状态机反馈目标可信度。

## 对论文写作的可用贡献点

1. **问题建模贡献**：明确提出“二维微偏移 -> 深度穿透 -> 三维控制灾难”的误差放大链路。  
2. **方法贡献**：提出“多帧滑窗 + DBSCAN + 稳定性阈值”的安全锁定机制。  
3. **工程贡献**：在无稳定ID条件下实现多目标关联与独立融合，提高实际部署可行性。  
4. **系统贡献**：将视觉不确定性显式转化为状态机可用的`locked`事件，形成感知到控制的闭环安全接口。  

## 当前方案边界与后续优化方向

- 当果串间距离过近且同步摆动时，最近邻关联可能出现短时误关联，可引入速度约束或匈牙利匹配优化。  
- 当深度噪声呈持续性结构化偏差时，仅依赖空间密度可能不足，可叠加时序模型（如卡尔曼）做二级平滑。  
- 当前阈值为经验-实验联合标定，可在后续通过自动超参数搜索获得跨园区泛化参数。  
- 可进一步引入任务约束（机械臂可达性、避障代价）作为候选目标重排序准则，提升端到端成功率。

## 论文可直接引用版（通俗 + 专业）

### 1) 研究问题（为什么要做）

本课题面向眼在手上机械臂的龙眼采摘任务。系统需要先找到果梗的三维坐标，再将该坐标下发给机械臂执行逼近与抓取。  
看起来这是一个“视觉定位”问题，但在真实果园环境中，它更像一个“安全控制输入”问题：只要坐标错一次，机械臂就可能朝错误方向运动，造成抓取失败甚至碰撞。

实际问题的触发链路很清晰：二维关键点在图像上只偏几像素，深度采样就可能从果梗表面跳到后方叶片，导致 $Z$ 值突变。  
也就是说，**小像素偏差会放大成大空间误差**。如果只信任单帧坐标，系统稳定性会明显不足。

### 2) 工程必要性（为什么必须做）

在果园这种非结构化场景中，光照变化、叶片摆动、遮挡切换都会引入短时随机误差。  
因此，单帧坐标往往“看起来像对的”，但并不一定“可直接用于控制”。

从工程角度看，多帧聚类模块是必需的，原因有三点：

1. 它把“偶然的错误帧”当作噪声剔除，而不是直接进入控制链路。  
2. 它不仅输出坐标，还输出稳定性指标（$\sigma$），让状态机有明确的安全判据。  
3. 它的计算代价远低于一次错误动作带来的试错成本（停机、损伤、碰撞风险）。

### 3) 算法选择逻辑（为什么是 DBSCAN）

本问题的观测数据具有一个典型特征：  
真实果梗点会在多帧中形成“高密度小簇”，穿透点则是“低密度散点”。  
这与 DBSCAN 的密度聚类假设天然一致。

相比常见方案：

- 平均/中值滤波：实现简单，但无法显式识别离群点，容易被穿透点拖偏。  
- KMeans：需要预先指定簇数，对离群点较敏感，不适合动态噪声环境。  
- Kalman 类方法：适合连续状态估计，但对突发型深度穿透的抑制不如密度剔除直接。

因此，本系统采用 DBSCAN 作为主去噪器：  
**先做密度判别剔除离群，再做质心融合输出稳定坐标**。

### 4) 方法流程（做了什么）

系统按“滑窗观测 -> 密度剔噪 -> 稳定判定”执行：

1. 构建长度为 $N=15$ 的观测滑窗，累积每帧果梗 `xyz`。  
2. 在窗口内运行 DBSCAN（$\epsilon=15$ mm，$MinPts=8$）。  
3. 将 `Noise` 点剔除，保留最大点数簇作为核心簇 $C_{core}$。  
4. 计算核心簇质心 $P_{final}$ 作为最终采摘点。  
5. 计算簇内离散度 $\sigma$，当 $\sigma < 3$ mm 时触发 `locked=True`。

这个流程的核心思想是：  
**不追求“第一帧就输出”，而追求“输出时可解释且可控制”**。

### 5) 多目标扩展（真实场景需要）

实际作业时，相机往往能同时看到多串龙眼。视觉模块不一定提供稳定 ID，因此系统加入了多目标轨迹管理：

- 用最近邻 + 距离门限进行每帧数据关联；  
- 每条轨迹独立维护一个 DBSCAN 融合器；  
- 通过短时失配容忍机制处理临时漏检。

这样做可以保证“每串果的点只和自己融合”，避免跨目标污染。

### 6) 参数解释（便于论文复现实验）

- $N=15$：兼顾稳定性与响应速度；  
- $\epsilon=15$ mm：覆盖果梗局部抖动与测量误差；  
- $MinPts=8$：要求超过半数帧支持同一簇；  
- $\sigma_{th}=3$ mm：作为目标锁定安全阈值；  
- 关联门限 80 mm：用于无 ID 条件下保持轨迹连续性。

这些参数并非“拍脑袋”，而是结合任务尺度、传感器噪声与机械臂控制容差进行的工程标定。

### 7) 应用价值（最终带来什么）

该方法在系统层面的价值可概括为一句话：  
**把不稳定视觉观测，变成可直接用于控制的稳定目标输入。**

具体收益包括：

- 降低深度穿透导致的误抓率；  
- 降低错误逼近造成的碰撞风险；  
- 为状态机提供可解释的锁定条件；  
- 提升复杂果园场景中的连续作业稳定性。

### 8) 一段可直接放在论文中的总结

针对眼在手上机械臂在龙眼采摘中易受深度穿透干扰的问题，本文提出一种多帧密度聚类定位方法。该方法通过构建时序滑窗并引入 DBSCAN 对三维观测点进行密度判别，有效剔除由关键点偏移引起的背景离群深度；随后对核心簇进行质心融合并结合方差阈值进行收敛判定，最终输出可用于控制的稳定采摘点坐标。进一步地，在视觉模块无稳定目标 ID 的条件下，系统通过最近邻关联实现多目标并行融合，使方法能够适配真实果园多果串场景。实验与工程实践表明，该策略可显著提升目标锁定稳定性与作业安全性，为果园机器人从“可检测”走向“可稳定执行”提供了关键支撑。
